Angular FACE SSR Comparisons

This app mirrors the Angular client-side baseline, but runs through Angular SSR so the initial HTML can be inspected before the custom elements upgrade on the client. The JSON panels still compare Angular's reactive-form model with the browser's current native FormData from the same forms after hydration.

Checkbox Control

This setup is valid because checkbox-like controls are driven by checked, not a scalar text-like value. When the box is unchecked, the field disappears from native FormData entirely. Angular has a dedicated accessor for native checkboxes for exactly this reason, which is why the mismatch is clearest here.

Multi-Select Control

This setup is valid because a multi-select submits a collection. The native shape is repeated powerUps entries, not one scalar value. Because this control is still select-like, it keeps both input and change rather than forcing a change-only contract. The mismatch is that Angular still keeps only one scalar string.

Calendar Picker

This setup is valid because choosing a date from a calendar grid is a commit action. There is no incremental text edit stream, so change is the meaningful event and omitting input is a reasonable implementation choice. That makes it a useful repro for Angular's default text-like assumptions.
